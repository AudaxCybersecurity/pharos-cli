# Security Policy

## Supported Versions

Pharos is currently in early release. Security fixes are applied to the latest version on the `main` branch.

## Reporting a Security Issue

Please do not open public GitHub issues for sensitive security reports.

To report a security concern, contact Audax Cybersecurity through the official business communication channels listed on the company website or GitHub organization profile.

When reporting, include as much detail as possible:

- Affected component or file
- Reproduction steps
- Expected behavior
- Actual behavior
- Potential impact
- Suggested fix, if available

## Scope

Security reports are welcome for:

- Unsafe command behavior
- Dependency security issues
- Incorrect handling of output files
- Bugs that could expose sensitive local information
- Issues that could mislead users during defensive assessment

Out of scope:

- Requests to add offensive exploitation features
- Social engineering requests
- Reports involving unauthorized scanning of third-party networks
- Findings from environments where testing was not authorized

## Responsible Use

Pharos is intended for authorized internal visibility, asset inventory, lab use, and defensive assessment workflows.

Use it only in environments where you have permission.
